# BUG-002 — Форма регистрации принимает невалидные данные (отсутствует валидация)

| Параметр | Значение |
|---|---|

| Модуль | Регистрация |
| Среда | Яндекс Браузер 26.4.5.634 (64-bit), Windows 10 22H2 |
| Стенд | prod (https://automationexercise.com/) |
| Severity | Major |
| Priority | Medium |
| Статус | Open |
| Автор | Викторов М.С. |
| Дата | 25.06.2026 |

## Описание

Форма регистрации (Create Account) принимает невалидные данные в полях и успешно создаёт аккаунт. Валидация формата полей отсутствует — как на клиенте, так и на сервере.

## Шаги воспроизведения

1. Открыть https://automationexercise.com/
2. Нажать "Signup / Login"
3. В форме "New User Signup!" ввести Name и Email, нажать "Signup"
4. В форме "Create Account" заполнить обязательные поля невалидными данными:
   - Name / First name / Last name — цифры и спецсимволы (например, `123!@#`)
   - Mobile Number — буквы (например, `abcdef`)
5. Нажать "Create Account"

## Фактический результат

Регистрация проходит успешно: появляется сообщение "Account Created!", аккаунт создаётся с невалидными данными.

## Ожидаемый результат

Поля валидируются согласно их типу (имя — буквы, телефон — только цифры). При вводе невалидных данных регистрация не проходит, рядом с полями выводятся сообщения об ошибках.

## Дополнительная информация

Проверено в DevTools (вкладка Network): форма отправляется POST-запросом (тип `document`), в ответ сервер возвращает страницу успешной регистрации, а не ошибку валидации. Следовательно, серверная валидация формата полей отсутствует. Поскольку клиентскую валидацию можно обойти, отсутствие серверной валидации создаёт риск попадания некорректных данных в систему и потенциальные проблемы безопасности (например, внедрение скриптов).
