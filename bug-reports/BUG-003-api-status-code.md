## Описание

При некорректных запросах API возвращает HTTP-статус 200 (успех), хотя фактически произошла ошибка. Реальный код результата передаётся только в теле ответа в поле `responseCode`.

## Шаги воспроизведения

1. Открыть Postman
2. Создать POST-запрос на `https://automationexercise.com/api/searchProduct`
3. Не передавать обязательный параметр `search_product` (отправить запрос с пустым телом)
4. Нажать "Send"

## Фактический результат

HTTP-статус ответа — **200 (OK)**. В теле ответа:
```json
{
    "responseCode": 400,
    "message": "Bad request, search_product parameter is missing in POST request."
}
```

## Ожидаемый результат

HTTP-статус ответа должен соответствовать фактическому результату — **400 (Bad Request)**, так как запрос некорректен (отсутствует обязательный параметр).

## Дополнительная информация

Дефект носит системный характер — воспроизводится на нескольких эндпоинтах (`/api/searchProduct`, `/api/verifyLogin`): при любой ошибке возвращается HTTP 200, а реальный код передаётся в теле.

Ошибка корректно сообщается в теле ответа (есть `responseCode` и понятное `message`), поэтому функциональность не нарушена — нарушена только HTTP-семантика (статус-код должен отражать результат запроса). Риск: системы и клиенты, ориентирующиеся на HTTP-статус, не распознают ошибку и воспримут запрос как успешный.
