# QA-проект: тестирование интернет-магазина Automation Exercise
 
Учебный проект по ручному и автоматизированному тестированию веб-приложения. Демонстрирует полный цикл работы тестировщика: от планирования и проектирования проверок до поиска, локализации и документирования дефектов, а также автоматизации API-тестов.
 
**Объект тестирования:** [Automation Exercise](https://automationexercise.com/) — демонстрационный интернет-магазин (UI + публичный API).
 
**Автор:** Викторов М.С.
 
---
 
## Инструменты и навыки
 
- **Ручное тестирование** — функциональное, UI, тестирование валидации, исследовательское
- **API-тестирование** — Postman (запросы, автоматические проверки)
- **Автоматизация** — Python, requests, pytest (API-автотесты)
- **Локализация дефектов** — Browser DevTools (Network, Console, Elements)
- **Техники тест-дизайна** — классы эквивалентности, анализ граничных значений
- **Документация** — тест-план, чек-листы, тест-кейсы, баг-репорты
- **Прочее** — Git / GitHub, Markdown

## Объём выполненной работы
 
- **Тест-план** — определён объём тестирования (5 модулей + API), подход, критерии, риски
- **Чек-листы** — 5 модулей, проверки сгруппированы по типам (позитивные / негативные / UI)
- **Тест-кейсы** — модуль «Регистрация»: 9 кейсов (позитивные пошаговые + негативные data-driven с применением техник тест-дизайна)
- **API-тестирование (Postman)** — коллекция из 6 запросов (3 эндпоинта), позитивные и негативные сценарии, автоматические проверки статус-кодов, тела и содержимого ответов
- **API-автотесты (Python)** — 7 тестов на pytest, покрывают те же эндпоинты позитивными и негативными сценариями
- **Баг-репорты** — 3 задокументированных дефекта разного типа

## Примечание
 
Формальные требования (ТЗ) к тестируемому приложению отсутствуют. Ожидаемые результаты в тест-кейсах и баг-репортах основаны на общепринятых стандартах и здравом смысле; спорные случаи отмечены как требующие уточнения у аналитика.
